# Part 2: Design

## 1. What overall style will best represent you?

The website will follow a minimalist, modern, and professional design style. The design will use a clean structure, strong functionality, and minimal visual clutter. It will prioritize usability and readability while maintaining a polished appearance.

## 2. What color scheme will you use and why?

The website will use the following professional color palette:

* White or light gray for the main background
* Navy blue as the primary accent color
* Steel blue as a secondary accent color
* Dark gray for body text
* Blue for links and call-to-action buttons

Blue is associated with trust, technology, stability, and professionalism. The light background will improve readability, while the darker accent colors will create clear visual hierarchy.

## 3. What fonts will you use for headings and body text?

* **Poppins** will be used for headings.
* **Inter** will be used for body text.

Poppins creates a modern and structured appearance, while Inter provides strong readability across desktop and mobile devices.

## 4. How will your design reflect your personality or field?

The design will reflect my background as both a military veteran and a software engineer by emphasizing structure, discipline, reliability, and efficiency. Clean layouts, organized sections, and practical navigation will communicate professionalism and technical competence without unnecessary visual complexity.

## 5. What layout will your homepage follow?

The homepage will follow this structure:

1. Navigation bar
2. Hero section
3. About section
4. Skills section
5. Projects section
6. Experience section
7. Contact section
8. Footer

The hero section will be the first major section visitors see. It will include my name, professional identity, a short introduction, and primary buttons such as **View Projects** and **Download Resume**.

## 6. How will you organize project sections visually?

Projects will be displayed using a responsive card-based layout. Each project card will include:

* Project screenshot
* Project title
* Short description
* Technologies used
* GitHub repository link

This layout will allow visitors to scan and compare projects quickly.

## 7. Will the site be mobile-friendly? How will you ensure responsiveness?

Yes. The website will be fully responsive and designed for desktop, tablet, and mobile devices.

Responsiveness will be implemented using:

* CSS Flexbox
* CSS Grid
* Relative sizing units
* Flexible images
* Media queries
* A responsive mobile navigation menu

## 8. What visual hierarchy will guide visitors?

Visual hierarchy will be established through font size, spacing, color contrast, alignment, and section organization.

Visitors will first see:

1. My name and professional identity
2. Primary call-to-action buttons
3. Professional summary
4. Skills and technologies
5. Featured projects
6. Experience and contact information

## 9. How will consistency be maintained across pages?

Consistency will be maintained through:

* A unified color palette
* Consistent typography
* Reusable button styles
* Standard section spacing
* Consistent project cards
* A common navigation structure
* Repeated heading and link styles

CSS variables will be used to store colors, spacing values, and other reusable design properties.

## 10. How will accessibility be considered?

Accessibility will be supported through:

* Sufficient color contrast
* Readable font sizes
* Semantic HTML elements
* Alternative text for images
* Labels for contact form fields
* Keyboard-accessible links and controls
* Visible focus states
* Responsive layouts
* Descriptive link and button text

The website will not rely only on color to communicate important information.

## 11. Will you use icons, images, or illustrations? Why?

Yes. The website will use:

* Technology icons
* Social media icons
* Project screenshots
* A professional profile image
* A downloadable resume icon or button

Images and icons will be stored locally in the project’s `images` directory whenever possible. These visual elements will make technical information easier to scan and will improve the professional presentation of the website.

## 12. What portfolio websites inspired your design?

The design is inspired by:

* Modern software engineering portfolio websites
* GitHub profile and repository pages
* LinkedIn professional profiles
* Minimalist technology websites

The main ideas taken from these sources are simplicity, readable content, clear professional identity, and easy access to projects and contact information.

# Part 3: Interactivity

## 1. What interactive elements will your site include?

The website will include:

* Responsive navigation menu
* Call-to-action buttons
* Project links
* GitHub and LinkedIn links
* Hover effects
* Contact form
* Smooth section navigation
* Scroll-to-top button
* Mobile menu

## 2. Will your site include a contact form? How will it work?

Yes. The website will include a contact form with fields for:

* Name
* Email address
* Message

For this project, the form will function as a front-end demonstration and will not send information to a real server. JavaScript will validate the fields and display a confirmation message after a successful submission.

## 3. What JavaScript features will you implement?

JavaScript will be used for:

* Responsive mobile navigation
* Smooth scrolling between sections
* Contact form validation
* Form confirmation messages
* Active navigation highlighting
* Scroll-to-top functionality
* Automatic display of the current year in the footer

## 4. How will users receive feedback from interactions?

Users will receive feedback through:

* Button and link hover effects
* Visible focus states
* Active navigation styles
* Contact form error messages
* Successful submission confirmation
* Mobile menu state changes
* Visual response when interactive controls are selected

## 5. How does interactivity improve the user experience?

Interactivity makes the website easier and more intuitive to use. Smooth navigation helps visitors reach important information quickly. Form validation prevents common errors, responsive menus improve mobile usability, and visual feedback makes clickable elements and successful actions clear.
